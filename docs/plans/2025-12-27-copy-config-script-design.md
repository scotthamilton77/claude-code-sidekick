# Design: scripts/copy-config.sh

Script to generate `.sidekick/sidekick.config` with verbose comments.

## Usage

```bash
# Include everything
scripts/copy-config.sh --all

# Include specific domains
scripts/copy-config.sh --core --llm

# Include specific features
scripts/copy-config.sh --features.reminders --features.statusline

# Combine as needed
scripts/copy-config.sh --core --features.reminders

# Output to different location
scripts/copy-config.sh --all --output ~/.sidekick/sidekick.config
```

## Flags

| Flag | Description |
|------|-------------|
| `--all` | Include all sections |
| `--core` | Core settings (logging, paths, supervisor, ipc) |
| `--llm` | LLM provider settings |
| `--transcript` | Transcript processing settings |
| `--features.reminders` | Reminders feature |
| `--features.statusline` | Statusline feature |
| `--features.session-summary` | Session summary feature |
| `--output <path>` | Override output path (default: `.sidekick/sidekick.config`) |
| `--help` | Show usage |

## Behavior

- **Missing directory**: Create `.sidekick/` automatically
- **Existing file**: Prompt to overwrite (interactive)
- **Comments**: Verbose with full descriptions per setting
- **Value source**: Read from `assets/sidekick/defaults/*.yaml` via `yq`

## Output Format

```bash
# ==============================================================================
# Sidekick Configuration
# Generated by: scripts/copy-config.sh
# Override values as needed - see docs/design/CONFIG-SYSTEM.md for details
# ==============================================================================

# ------------------------------------------------------------------------------
# Core Settings
# Logging, paths, supervisor process, and IPC configuration
# ------------------------------------------------------------------------------

# Log level: debug, info, warn, error
# debug = all messages including internal state
# info = normal operation messages
# warn = potential issues that don't prevent operation
# error = failures and critical issues only
core.logging.level=info

# Output format: pretty (human-readable) or json (structured for parsing)
core.logging.format=pretty

# ... (continues for all settings)
```

## Dependencies

- `yq` - YAML parser for reading defaults files
- Script checks for `yq` and errors with install instructions if missing

## Implementation

1. Parse arguments - build list of sections to include
2. Check output path - create directory if needed, prompt if file exists
3. Generate content - for each section, emit header + commented settings
4. Write file - output to target path
