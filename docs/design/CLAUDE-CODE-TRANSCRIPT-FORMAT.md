# Claude Code Transcript Format

Reference document for parsing Claude Code session transcripts.
Reverse-engineered from real transcripts (2026-03-14, Claude Code v2.1.60–2.1.75).

## File Layout

Transcripts live at `~/.claude/projects/{projectId}/`. Two layouts exist:

### Simple (older sessions)
```
~/.claude/projects/{projectId}/{sessionId}.jsonl
```

### Directory (sessions with subagents)
```
~/.claude/projects/{projectId}/{sessionId}/
  {sessionId}.jsonl                     # Parent conversation
  subagents/
    agent-{agentId}.jsonl               # Each subagent's full transcript
    agent-{agentId}.meta.json           # { agentType, worktreePath }
  tool-results/
    {hash}.txt                          # Large tool outputs stored externally
```

The `projectId` matches Sidekick's registry ID (e.g., `-Users-scott-src-projects-foo`).

## Entry Format

Each line is a JSON object. All entries share common metadata fields:

| Field | Type | Presence | Description |
|-------|------|----------|-------------|
| `type` | string | Always | Entry category (see taxonomy below) |
| `uuid` | UUID | Most | Unique entry identifier |
| `timestamp` | ISO 8601 | Always | When the entry was created |
| `sessionId` | UUID | Most | Session grouping |
| `parentUuid` | UUID/null | Most | Parent message reference |
| `isSidechain` | boolean | Most | Sidechain (e.g., SessionStart hook) vs. main conversation |
| `isMeta` | boolean | ~10% | System-injected metadata, not user-visible |
| `isCompactSummary` | boolean | Rare | Entry generated by compaction |
| `userType` | string | Most | `"external"` or `"agent"` |
| `version` | semver | Most | Claude Code version |
| `gitBranch` | string | Most | Repo branch at entry time |
| `cwd` | path | Most | Working directory |
| `slug` | string | Recent | Session display name |
| `toolUseID` | string | ~60% | Associated tool invocation |
| `parentToolUseID` | string | ~60% | Parent tool in chain |

## Entry Type Taxonomy

### Conversation Entries

#### `user` — User Messages
```jsonl
{"type":"user","uuid":"...","timestamp":"...","message":{"role":"user","content":"..."},"isMeta":false,"promptId":"...","permissionMode":"..."}
```

**Content can be:**
- `string` — Plain text prompt
- `Array<ContentBlock>` — Mixed content (text + tool_result blocks)

**Content block types in user messages:**
- `{ type: "text", text: "..." }` — User text
- `{ type: "tool_result", tool_use_id: "...", content: "...", is_error: boolean }` — Tool execution result

**Notable fields:** `promptId`, `permissionMode`, `sourceToolAssistantUUID`

#### `assistant` — Model Responses
```jsonl
{"type":"assistant","uuid":"...","timestamp":"...","message":{"role":"assistant","content":[...],"model":"...","usage":{...}},"requestId":"...","stop_reason":"..."}
```

**Content block types in assistant messages:**
- `{ type: "text", text: "..." }` — Response text
- `{ type: "tool_use", id: "...", name: "...", input: {...} }` — Tool invocation
- `{ type: "thinking", thinking: "...", signature: "..." }` — Extended thinking

**Notable fields:** `model`, `stop_reason` ("end_turn" | "tool_use"), `requestId`

**Usage metadata** (in `message.usage`):
```json
{
  "input_tokens": 500,
  "output_tokens": 200,
  "cache_creation_input_tokens": 100,
  "cache_read_input_tokens": 300,
  "service_tier": "pro",
  "model": "claude-opus-4-6"
}
```

### System Entries

#### `system` — System Events

| Subtype | Frequency | Description | Key Fields |
|---------|-----------|-------------|------------|
| `compact_boundary` | ~15/session | Conversation compacted | `compactMetadata.trigger`, `compactMetadata.preTokens`, `logicalParentUuid` |
| `turn_duration` | ~60/session | Elapsed time for one turn | `durationMs` (integer ms) |
| `stop_hook_summary` | ~100/session | Hook execution summary | `hookCount`, `hookInfos[]` (command, durationMs), `hookErrors[]` |
| `local_command` | ~20/session | Local command markers | `content` (HTML/text), `level` (info/warning) |
| `api_error` | Rare | API retry metadata | `error`, `retryInMs`, `retryAttempt`, `maxRetries` |

### Progress Entries

#### `progress` — Execution Progress

Nested `data.type` distinguishes subtypes:

| data.type | Description | Key Fields |
|-----------|-------------|------------|
| `hook_progress` | Hook execution | `hookEvent` (PreToolUse, PostToolUse, SessionStart, SessionEnd, Stop), `hookName`, `command` |
| `agent_progress` | Subagent activity | `agentId`, `message` (wraps subagent entries) |
| `mcp_progress` | MCP server tool | `status`, `serverName`, `toolName` |
| `bash_progress` | Long-running bash | `output`, `fullOutput`, `elapsedTimeSeconds`, `totalLines` |

### Metadata Entries

| Type | Frequency | Description |
|------|-----------|-------------|
| `queue-operation` | Very high (99%+) | Internal message queuing — **skip in UI** |
| `file-history-snapshot` | ~150/session | File tracking checkpoints — **skip in UI** |
| `last-prompt` | Rare | Last prompt cache — **skip in UI** |
| `pr-link` | Rare | PR metadata: `prNumber`, `prUrl`, `prRepository` |

## Subagent Linking Chain

When Claude Code spawns a subagent via the Agent tool:

```
1. Parent transcript: assistant entry with tool_use { name: "Agent", id: "toolu_..." }
   └─ input: { description, prompt, isolation, mode, subagent_type }

2. Parent transcript: progress entries with data.type: "agent_progress"
   └─ data.agentId: "a360d2cb0bb10ddaf"
   └─ parentToolUseID: "toolu_..." (links back to step 1)
   └─ Wraps every subagent message/action as they stream

3. Subagent transcript: subagents/agent-{agentId}.jsonl
   └─ Every entry has agentId field
   └─ sessionId references the parent session
   └─ Same entry format as parent (user, assistant, progress, etc.)

4. Subagent metadata: subagents/agent-{agentId}.meta.json
   └─ { agentType: "Explore"|"general-purpose"|..., worktreePath: "..." }
```

**Reconstruction:** To display a subagent's work, read `subagents/agent-{agentId}.jsonl`. To find which subagents exist for a session, either scan the `subagents/` directory or follow `agent_progress` entries in the parent transcript.

## UI Rendering Recommendations

### Render in Transcript Panel
| Entry | TranscriptLine type | Display |
|-------|-------------------|---------|
| `user` (text) | `user-message` | User prompt text |
| `user` (tool_result) | `tool-result` | Tool output, error flag |
| `assistant` (text) | `assistant-message` | Response + collapsible thinking |
| `assistant` (tool_use) | `tool-use` | Tool name + input preview |
| `system/compact_boundary` | `compaction` | Compaction marker with preTokens |
| `system/turn_duration` | `turn-duration` | Elapsed time badge |
| `system/api_error` | `api-error` | Retry indicator |
| `pr-link` | `pr-link` | Clickable PR reference |
| `isSidechain: true` entries | Same types, visually marked | Sidechain indicator |

### Skip
| Entry | Reason |
|-------|--------|
| `queue-operation` | 99%+ of entries, internal queuing noise |
| `file-history-snapshot` | Internal bookkeeping |
| `last-prompt` | Cache artifact |
| `progress/hook_progress` | Sidekick captures hooks separately |
| `progress/bash_progress` | Final tool_result has the content |
| `progress/mcp_progress` | Low value for now |
| `progress/agent_progress` | Streaming wrapper; subagent transcript has the real content |
| `system/stop_hook_summary` | Internal hook timing |
| `system/local_command` | Usually empty markers |

### Capture as Metadata (not rendered as lines)
| Field | Use |
|-------|-----|
| `model` | Show which model responded |
| `stop_reason` | Understand turn flow |
| `isCompactSummary` | Flag summarized-not-real messages |
| `message.usage` | Token metrics for cost/context analysis |
